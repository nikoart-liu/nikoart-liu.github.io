Rust - 泛型、Trait 和生命周期#

知识概览#

  • 本章核心主题：通过抽象机制消除代码重复，实现灵活且安全的代码
  • 重要程度：⭐⭐⭐（核心概念，后续章节的基础）
  • 预计学习时间：2-3小时
  • 前置知识要求：
    • 基本语法（变量、函数、结构体、枚举）
    • 所有权和借用
    • 错误处理的基础（Option、Result）

核心概念#

泛型类型（Generics）#

定义：允许代码处理多种类型而非具体类型的抽象机制

官方文档：

url: https://doc.rust-lang.org/book/ch10-01-syntax.html
title: "Generic Data Types - The Rust Programming Language"
host: doc.rust-lang.org
favicon: favicon-de23e50b.svg

语法：

// 函数泛型
fn largest<T: PartialOrd>(list: &[T]) -> &T {
    let mut largest = &list[0];
    for item in list {
        if item > largest {
            largest = item;
        }
    }
    largest
}

// 结构体泛型（单泛型）
struct Point<T> {
    x: T,
    y: T,
}

// 结构体泛型（多泛型）
struct Point<T, U> {
    x: T,
    y: U,
}

// impl 块
impl<T> Point<T> {
    fn x(&self) -> &T { &self.x }
}

// 为特定类型实现方法
impl Point<f32> {
    fn distance_from_origin(&self) -> f32 {
        (self.x.powi(2) + self.y.powi(2)).sqrt()
    }
}

性能：通过单态化在编译时生成具体类型代码，零运行时开销

Trait#

定义：定义共享行为的抽象机制，类似其他语言的接口

官方文档：

url: https://doc.rust-lang.org/book/ch10-02-traits.html
title: "Defining Shared Behavior with Traits - The Rust Programming Language"
host: doc.rust-lang.org
favicon: favicon-de23e50b.svg

语法：

// 定义 Trait
pub trait Summary {
    // 抽象方法（必须实现）
    fn summarize(&self) -> String;

    // 默认实现（可选重写）
    fn summarize_author(&self) -> String {
        format!("Anonymous")
    }
}

// 为类型实现 Trait
pub struct NewsArticle {
    pub headline: String,
    pub location: String,
}

impl Summary for NewsArticle {
    fn summarize(&self) -> String {
        format!("{}, by {}", self.headline, self.location)
    }
}

Trait Bound 作为参数：

// 方法 1：impl Trait（简洁）
pub fn notify(item: &impl Summary) {
    println!("Breaking news: {}", item.summarize());
}

// 方法 2：Trait Bound（灵活）
pub fn notify<T: Summary>(item: &T) {
    println!("Breaking news: {}", item.summarize());
}

// 多个 Trait
pub fn notify<T: Summary + Display>(item: &T) { ... }

// where 子句（更清晰）
fn some_function<T, U>(t: &T, u: &U) -> i32
where
    T: Display + Clone,
    U: Clone + Debug,
{
    // ...
}

返回类型：

// 只能返回一种具体类型（静态分发）
fn returns_summarizable() -> impl Summary {
    NewsArticle { /* ... */ }
}

// 返回动态类型（有运行时开销）
fn get_content(condition: bool) -> Box<dyn Summary> {
    if condition {
        Box::new(NewsArticle { /* ... */ })
    } else {
        Box::new(Tweet { /* ... */ })
    }
}

有条件实现：

struct Pair<T> { x: T, y: T }

impl<T: Display + PartialOrd> Pair<T> {
    fn cmp_display(&self) {
        if self.x >= self.y {
            println!("x = {} is larger", self.x);
        } else {
            println!("y = {} is larger", self.y);
        }
    }
}

生命周期（Lifetimes）#

定义：确保引用有效性、防止悬垂引用的编译时机制

官方文档：

url: https://doc.rust-lang.org/book/ch10-03-lifetime-syntax.html
title: "Validating References with Lifetimes - The Rust Programming Language"
host: doc.rust-lang.org
favicon: favicon-de23e50b.svg

语法：

// 函数生命周期标注
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}

// 结构体生命周期
struct ImportantExcerpt<'a> {
    part: &'a str,
}

生命周期省略规则：

  1. 每个引用参数获得独立生命周期
  2. 若只有一个输入生命周期，返回值使用相同生命周期
  3. 若有 &self 或 &mut self，返回值使用 self 的生命周期

'static 生命周期：表示引用在整个程序运行期间有效（用于字符串字面量）

let s: &'static str = "hello";  // 字符串字面量

// 错误示例：不能将非静态数据标注为 'static
fn error_example() -> &'static str {
    let s = String::from("world");
    &s  // ❌ 编译错误
}

综合实践#

题目：通用时间线结构#

//  综合应用题：编写一个通用的"最长时间线"结构
//   要求：
//   1. 能够存储事件和它们的时间戳
//   2. 提供查找最新事件的方法
//   3. 提供格式化输出的能力
//   4. 能够比较两个时间线
use std::{cmp::Ordering, cmp::Ord, fmt::Display, fmt::Formatter, fmt::Result};
fn main() {
    let mut tl1 = Timeline {
        name: String::from("Project A"),
        events: vec![(1, String::from("启动")), (3, String::from("里程碑"))],
    };
    let mut tl2 = Timeline {
        name: String::from("Project B"),
        events: vec![(100, String::from("启动")), (500, String::from("完成"))],
    };
    // 测试 Display
    println!("{}", tl1);
    println!("{}", tl2);
    if let Some((time, event)) = tl1.latest_event() {
        println!("\n最新事件 ({}): 时间={}, 内容={}", tl1.name, time, event);
    }
    if tl2 > tl1 {
        println!("\n{} 更新", tl2.name);
    }
}

struct Timeline<T: Ord + Display> {
    name: String,
    events: Vec<(T, String)>,
}

impl<T: Ord + Display> Timeline<T> {
    // 查找最新事件（时间戳最大的）
    fn latest_event(&self) -> Option<&(T, String)> {
        self.events.iter().max_by_key(|e| &e.0)
    }
}
impl<T: Ord + Display> Display for Timeline<T> {
    fn fmt(&self, f: &mut Formatter<'_>) -> Result {
        write!(f, "Timeline: {}\nEvents:\n", self.name)?;
        for (time, event) in &self.events {
            writeln!(f, "  {}: {}", time, event)?;
        }
        Ok(())
    }
}

impl<T: Ord + Display> PartialOrd for Timeline<T> {
    fn partial_cmp(&self, other: &Self) -> Option<Ordering> {
        match (self.latest_event(), other.latest_event()) {
            (Some((t1, _)), Some((t2, _))) => t1.partial_cmp(t2),
            _ => None,
        }
    }
}

// 为了使用 >= 等操作符
impl<T: Ord + Display> PartialEq for Timeline<T> {
    fn eq(&self, other: &Self) -> bool {
        match (self.latest_event(), other.latest_event()) {
            (Some((t1, _)), Some((t2, _))) => t1 == t2,
            _ => false,
        }
    }
}

关键要点#

  • 泛型 + Trait 约束 = 有用的泛型（否则泛型"什么都做不了"）
  • 生命周期标注用于消除歧义，编译器通过省略规则自动推导
  • impl Trait 与 Box<dyn Trait>：静态分发（零开销） vs 动态分发（运行时查表）
  • 单态化确保泛型代码零运行时开销
  • Trait 遵循孤儿规则：只能为本地类型或 trait 实现
  • 字符串字面量自动拥有 'static 生命周期
  • PartialOrd 必须与 PartialEq 成对实现
