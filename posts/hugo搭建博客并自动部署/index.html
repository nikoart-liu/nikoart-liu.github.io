Hugo 搭建博客并部署到 Github Pages 指南

January 29, 2026 · 3 min · 588 words · nikoart

一、环境准备及 Hugo 安装#

安装 Homebrew#

如果你还没有安装 Homebrew，请先打开终端执行以下命令来安装这个 macOS 上最常用的包管理器：

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

使用 Homebrew 安装 Hugo#

Homebrew 安装包的方式非常简洁，一条命令即可完成：

brew install hugo

安装完成后，验证 Hugo 是否正确安装：

hugo version

如果终端显示类似 hugo v0.155.0+extended+withdeploy darwin/arm64 BuildDate=2026-01-28T16:28:23Z VendorInfo=Homebrew 的版本信息，说明安装成功。

二、创建 Hugo 站点#

初始化站点#

使用 Hugo 命令创建一个全新的网站基础结构：

hugo new site my-blog --format=yaml
cd my-blog
git init

执行完成后，Hugo 会创建一个名为 my-blog 的目录。进入这个目录查看自动生成的文件结构：

my-blog/
├── archetypes/
│   └── default.md
├── content/
├── layouts/
├── static/
├── themes/
└── hugo.yaml

这个结构的设计非常清晰：

  • archetypes 用于定义文章模板
  • content 存放你的文章内容
  • layouts 控制页面布局
  • static 存放静态资源如图片
  • themes 用于放置主题
  • hugo.yaml 站点配置文件

启动 Hugo 的开发服务器以查看站点：

hugo server

配置站点信息#

打开 hugo.yaml，进行基本的站点配置：

baseURL: 'https://nikoart-liu.github.io/'
title: 'my blog'
theme: 'PaperMod'
languageCode: 'zh-CN'

params:
  author: 'author'
  description: ''
  defaultTheme: 'auto'
  ShowReadingTime: true
  ShowPostNavLinks: true
  ShowBreadCrumbs: true
  ShowWordCount: true
  ShowReadingProgress: true
  ShowScrollToTop: true

  homeInfoParams:
    Title: 'Hello'
    Content: ''

  socialIcons:
    - name: 'github'
      url: 'https://github.com/nikoart-liu'

  fuseOpts:
    isCaseSensitive: false
    shouldSort: true
    location: 0
    distance: 1000
    threshold: 0.4
    minMatchCharLength: 0
    keys: ["title", "permalink", "summary", "content"]

menu:
  main:
    - identifier: 'archives'
      name: '归档'
      url: '/archives/'
      weight: 5
    - name: "分类"
      url: "/categories/"
      weight: 10
    - identifier: 'tags'
      name: '标签'
      url: '/tags/'
      weight: 15
    - identifier: "search"
      name: "搜索"
      url: "/search/"
      weight: 20

outputs:
  home: ["HTML", "RSS", "JSON"]

安装主题#

上面的配置使用了 PaperMod 主题，可以使用 git submodule 的方式来安装：

git submodule add --depth=1 https://github.com/adityatelange/hugo-PaperMod.git themes/PaperMod
git submodule update --init --recursive # needed when you reclone your repo (submodules may not get cloned automatically)

更新，在你的站点目录下运行：

git submodule update --remote --merge

在站点配置文件中添加主题：

theme: ["PaperMod"]

添加搜索页：将以下内容添加到站点配置中，

outputs:
  home: ["HTML", "RSS", "JSON"]

在 content 目录中创建一个带有 search.md 的页面，包含以下内容：

---
title: "Search" # in any language you want
layout: "search" # necessary for search
# url: "/archive"
# description: "Description for Search"
summary: "search"
placeholder: "placeholder text in search input box"
---

三、发布到 Github Pages#

在 github 上创建两个仓库：

  • 源码仓库，用来对站点配置版本管理
  • page 仓库，管理 public 目录下的静态资源（仓库名必须是 <username>.github.io,并且是公开仓库）

配置 Action#

首先，需要申请一个 PAT（Personal access token），可以在 profile->settings->Developer settings->Personal access tokens->Token(classic) 创建，创建完成后复制，在源码仓库的 Settings->Secrets and variables->Actions->Repository secrets ，点击 New repository secret，保存token，Name 必须是 PAGES_REPO_TOKEN，与 workflow 中的配置一致。

在源码仓库中，选择 Actions，点击 New workflow 创建一个新的工作流，对应 .github/workflows/hugo.yml，填入一下内容：

name: Deploy Hugo site to Pages Repo

on:
  push:
    branches: ["master"]
  workflow_dispatch:

permissions:
  contents: read

concurrency:
  group: "pages"
  cancel-in-progress: false

defaults:
  run:
    shell: bash

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    env:
      HUGO_VERSION: 0.155.0
    steps:
      # 先 checkout，再安装依赖（顺序重要）
      - name: Checkout
        uses: actions/checkout@v4
        with:
          submodules: recursive
          fetch-depth: 0
          token: ${{ secrets.GITHUB_TOKEN }}  # 确保能拉取私有 submodule
      
      - name: Install Hugo CLI
        run: |
          wget -O ${{ runner.temp }}/hugo.deb https://github.com/gohugoio/hugo/releases/download/v${HUGO_VERSION}/hugo_extended_${HUGO_VERSION}_linux-amd64.deb \
          && sudo dpkg -i ${{ runner.temp }}/hugo.deb
          hugo version
      
      - name: Install Dart Sass
        run: sudo snap install dart-sass
      
      - name: Update theme submodule
        run: |
          git submodule update --remote --merge || echo "Using existing submodule commit"
          git submodule status
      
      - name: Install Node.js dependencies
        run: "[[ -f package-lock.json || -f npm-shrinkwrap.json ]] && npm ci || true"
      
      - name: Build with Hugo
        env:
          HUGO_CACHEDIR: ${{ runner.temp }}/hugo_cache
          HUGO_ENVIRONMENT: production
          HUGO_ENV: production
        run: |
          rm -rf public
          hugo \
            --gc \
            --cleanDestinationDir \
            --minify \
            --baseURL "https://nikoart-liu.github.io/"
          # 验证构建输出
          ls -la public/
      
      - name: Deploy to Pages Repository
        uses: peaceiris/actions-gh-pages@v4
        with:
          personal_token: ${{ secrets.PAGES_REPO_TOKEN }}
          external_repository: nikoart-liu/nikoart-liu.github.io
          publish_branch: master
          publish_dir: ./public
          commit_message: 'Auto deploy from source repo: ${{ github.event.head_commit.message }}'
          user_name: 'github-actions[bot]'
          user_email: 'github-actions[bot]@users.noreply.github.com'
          # 可选：强制推送（如果历史混乱）
          # force_orphan: true

当源码仓库推送变更后，会触发 Actions 执行操作。

源码仓库添加一个 .gitignore，避免将本地的public 目录提交到仓库。

public/

配置 Pages#

在 page 仓库的 Settings 中，点击左侧 Pages 菜单，在 Build and deployment 一栏中，将 Source 修改成 Deploy from a branch，Branch 选择正确的分支。

当源码仓库触发 Actions 之后，可以在此页面看见上次部署的时间。
